Спроектировать и реализовать шаблонный класс Container со следующими возможностями:
  - добавить новый элемент в конец;
  - удалить последний элемент;
  - получить элемент по индексу;
  - узнать кол-во элементов;
  - проверить на пустоту;
  - очистить контейнер.

Класс должен быть эффективным и универсальным (должен подходить для хранения
экземпляров любых типов и использования где угодно). Каждый метод должен иметь
сложность O(1) (без учёта возможной реаллокации).

Для тех, кто в танке: не надо реализовывать Container с помощью какого-либо
готового класса контейнера.
